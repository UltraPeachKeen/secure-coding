# 유지보수 및 수정 기록

| 단계 | 발견한 문제 | 원인 | 수정 | 재검증 |
|---|---|---|---|---|
| 설계 검토 | 신고된 상품을 물리 삭제하면 신고 감사 기록이 사라지거나 FK 오류 발생 | `reports`가 `products`를 참조 | 소유자 삭제를 `deleted` 상태의 소프트 삭제로 변경, 공개 목록/이미지 숨김 | 상품 삭제 테스트 통과 |
| 자동 테스트 | 신고 누적 테스트에서 한 신고자만 반복된 것처럼 관찰 | 테스트가 브라우저 세션의 내부 ID를 강제로 교체 | 실제 logout/login 경로로 사용자 전환 | 3인 신고 자동 차단·휴면 테스트 통과 |
| 배포 보안 | 고정 개발용 `SECRET_KEY`로도 서버가 실행 가능 | 편의용 기본값 | 환경변수 없으면 비테스트 실행 즉시 실패 | 설정 로딩 및 테스트 통과 |
| 검색 검토 | `%`, `_`가 검색 wildcard로 해석되어 의도보다 넓은 결과 가능 | LIKE 메타문자 미처리 | escape 문자를 명시하고 세 문자를 이스케이프 | SQL 형태 검색 테스트 통과 |
| 재현성 검증 | 한글 폴더에서 Compose 자동 프로젝트명이 비어 구성 실패 | 경로명 정규화 결과에 의존 | Compose 최상위 `name: tiny-market` 명시 | `docker compose config --quiet` 통과 |

향후 유지보수 우선순위는 Alembic migration, PostgreSQL 전환, Redis 기반 로그인/요청 rate limit, 관리자 메시지 moderation, 썸네일 생성과 악성 이미지 스캔이다.
