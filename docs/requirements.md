# 요구사항 분석

## 과제 근거

- 강의 슬라이드 34쪽: 요구사항 분석, 설계, 구현, 체크리스트/테스트, 유지보수 전 과정과 보안 약점 수정 내용을 보고서로 작성한다.
- 강의 슬라이드 35쪽: 가입, 상품 등록/조회, 사용자 소통, 악성 사용자·상품 차단, 사용자 송금, 상품 검색, 관리자 전체 관리를 최소 기능으로 명시한다.
- 디스코드 공지: 제출일은 슬라이드의 7월 23일이 아닌 2026년 7월 24일 23:59:59이며 LMS에 PDF 한 개를 제출한다.

## 기능 요구사항

| ID | 요구사항 | 완료 조건 |
|---|---|---|
| FR-01 | 회원가입/로그인/로그아웃 | 중복 계정을 거부하고 정상 세션을 생성·종료한다. |
| FR-02 | 프로필/마이페이지 | 프로필을 조회하고 본인의 소개글·비밀번호를 변경한다. |
| FR-03 | 상품 CRUD | 로그인 사용자가 상품을 만들고 본인 상품만 수정·삭제한다. |
| FR-04 | 상품 목록/상세/검색 | 공개 목록과 상세를 제공하고 제목·설명으로 검색한다. |
| FR-05 | 내 상품 관리 | 본인이 등록한 상품과 상태를 한 화면에서 관리한다. |
| FR-06 | 전체/1:1 채팅 | 전체 메시지와 지정 사용자 간 메시지를 저장·조회한다. |
| FR-07 | 사용자/상품 신고 | 신고 사유를 저장하고 같은 대상 중복 신고를 거부한다. |
| FR-08 | 자동 제재 | 유효 신고 3회 이상 상품은 차단, 사용자는 휴면 처리한다. |
| FR-09 | 가상 잔액 송금 | 잔액을 원자적으로 이동하고 거래 내역을 저장한다. |
| FR-10 | 관리자 | 사용자·상품·신고·거래·감사 로그를 조회/관리한다. |
| FR-11 | 샘플 데이터 | CLI로 관리자와 일반 사용자 및 예시 데이터를 생성한다. |

## 보안 요구사항

| ID | 요구사항 | 구현 방침 |
|---|---|---|
| SR-01 | 비밀번호 보호 | Werkzeug salted hash, 최소 10자, 변경 시 현재 비밀번호 확인 |
| SR-02 | SQL Injection 방어 | SQLAlchemy ORM/매개변수 바인딩만 사용 |
| SR-03 | XSS 방어 | Jinja 자동 이스케이프, `safe` 미사용, 길이 제한 |
| SR-04 | CSRF 방어 | Flask-WTF 전역 보호, 상태 변경은 POST |
| SR-05 | 인증/인가 | login_required, 소유권 검사, admin_required |
| SR-06 | 입력 검증 | WTForms와 모델 CHECK, 서버 측 대상/상태 검증 |
| SR-07 | 파일 업로드 | 2MB, PNG/JPEG/GIF/WebP, Pillow 검증, UUID 파일명 |
| SR-08 | 송금 안전성 | 양수·상한·잔액·자기송금 검사, 조건부 UPDATE와 단일 commit |
| SR-09 | 신고 무결성 | 복합 UNIQUE와 재확인, 자기 대상 신고 금지 |
| SR-10 | 휴면 제한 | 로그인 거부 및 기존 세션 쓰기 작업 차단 |
| SR-11 | 비밀 관리 | 환경변수, `.env.example`, DB/.env/uploads gitignore |
| SR-12 | 오류/헤더 | 일반 오류 페이지, CSP 등 기본 보안 헤더, 안전한 쿠키 |

## 해석 및 기본값

- "실시간 채팅"은 안정성을 위해 3초 AJAX polling으로 구현한다.
- "일정 횟수"는 기본 3회이며 `REPORT_THRESHOLD`로 조정한다.
- "누구나 상품 조회"는 비회원 공개 조회로 해석한다.
- 관리 범위는 사용자 상태, 상품 상태, 신고 처리, 거래/감사 기록 조회로 구체화한다.
- 가상 잔액은 원 단위 정수이며 기본 100,000원이다.
