# 보안 테스트 체크리스트

| ID | 공격/비정상 입력 | 기대 결과 | 자동 테스트 |
|---|---|---|---|
| S-01 | DB에서 비밀번호 확인 | 평문이 아닌 scrypt 해시 | O |
| S-02 | 검색에 `' OR 1=1 --` | SQL로 실행되지 않고 정상 응답 | O |
| S-03 | 상품/채팅에 `<script>`/이벤트 속성 | HTML로 이스케이프, 실행 안 됨 | O |
| S-04 | CSRF 토큰 없이 가입/삭제/송금 | HTTP 400 | O(가입) / 수동 |
| S-05 | 타인 상품 edit/delete URL | HTTP 403 | O |
| S-06 | 일반 사용자의 `/admin` 접근 | HTTP 403 | O |
| S-07 | `.exe`, 가짜 PNG, 초대형 파일 | 업로드 거부(400/413 또는 폼 오류) | O(가짜 PNG) |
| S-08 | 0원·음수·잔액 초과·자기 송금 | 거부, 잔액/내역 불변 | O |
| S-09 | 동일 대상 반복 신고 | 한 건만 저장 | O |
| S-10 | 자기 자신/자기 상품 신고 | HTTP 400 | 수동 |
| S-11 | 휴면 사용자 로그인/쓰기 | 로그인/기능 거부 | O(로그인) |
| S-12 | 외부 URL을 `next`로 전달 | 외부 redirect 안 함 | 코드 검토/수동 |
| S-13 | 오류/없는 ID 요청 | 내부 stack/DB 정보 미노출 | 수동 |
| S-14 | 응답 헤더 확인 | CSP/nosniff/frame/referrer/permissions 존재 | O |
| S-15 | `git ls-files`와 `git grep` | `.env`, DB, 업로드, 실제 키 없음 | 최종 검증 |
| S-16 | 두 송금 요청 동시 시도 | 총 차감이 잔액을 초과하지 않음 | 설계 검토/수동 부하 |
